# Task Core Directions

## Что изменилось

- Добавлен отдельный интерактивный дизайн-экран `public/concepts/task-core-directions.html`.
- Основное приложение не менялось.
- Экран содержит три архитектурных направления одного и того же task flow:
  - `A · Compact` — плотный классический task tracker;
  - `B · Editorial` — спокойная рабочая среда с большим редактором;
  - `C · Operator` — task tracker с точечной рекомендацией ассистента.
- Во всех направлениях можно выбрать задачу, завершить её, открыть меню параметров правой кнопкой или через кнопку, закрыть редактор и добавить новую задачу.
- В `B · Editorial` системные разделы отделены от пользовательских списков. Списки организованы внутри сворачиваемых областей `Работа`, `Личное`, `Здоровье` и получили компактные визуальные маркеры.
- В Editorial восстановлен верхний уровень продукта: постоянная module rail для `Задач`, `Календаря`, `Привычек`, `Фокуса`, `Заметок` и поиска. Навигация внутри Tasks находится во втором sidebar и больше не смешивается с модулями.
- Исправлено наложение длинных названий: sidebar использует однострочное сокращение с ellipsis, счётчики сохраняют отдельную колонку.
- Пользовательские списки в Editorial редактируются: название, одна из пяти иконок и один из четырёх цветовых маркеров.
- В дизайн-лабораторию добавлены палитры `Sage`, `Sky`, `Amber` и типографические режимы `Clean`, `Editorial type`.

## Почему

До следующего изменения основного приложения нужен выбор базовой архитектуры. Варианты сравнивают именно компоновку и роль detail pane, а не цвета одного и того же экрана.

## Риски

- Это дизайн-прототип: изменения не сохраняются в Supabase.
- Мобильная компоновка задана CSS breakpoint, но точную device screenshot-проверку нужно повторить после выбора направления.
- После выбора одного направления его нужно перенести в основное приложение и удалить конкурирующую старую архитектуру, а не наслаивать новый CSS.
