# MVP Reliability Pass

## 1. Что изменилось

- Навигация по модулю задач теперь помнит последний открытый раздел и возвращает пользователя в него после перехода в заметки, календарь или другой модуль.
- Focus Companion в актуальном интерфейсе получил рабочие обработчики режима, таймера, категории звука, воспроизведения и громкости.
- Нулевая громкость больше не сбрасывается при нормализации или перезагрузке состояния.
- Привычки сгруппированы по времени дня и показывают прогресс, серию и семь последних дней. Группу можно изменить прямо в строке привычки.
- Добавлена статическая reliability-проверка обязательных модулей и ключевых UI-контрактов.
- Версии клиентских ресурсов и service worker синхронно подняты до `v120`.

## 2. Почему

После нескольких крупных UI-итераций часть новых экранов использовала старые скрытые обработчики. В результате элементы выглядели готовыми, но не реагировали на действия. Этот проход связывает актуальную оболочку с единым состоянием и добавляет быструю проверку, которая ловит повторное исчезновение ключевых контрактов.

## 4. Автоматические проверки

Выполнено успешно:

```text
node --check public/app.js
node --check public/sw.js
node --check server.js
node scripts/reliability-check.mjs
npm run check
git diff --check
```

Reliability check контролирует наличие восьми модулей, память task view, обработчики Focus Companion, сохранение громкости, историю привычек и совпадение версий ресурсов.

## 5. Браузерная проверка

URL: `http://127.0.0.1:4174/?fresh=120`

Desktop, `1280 x 720`:

- горизонтальное переполнение: нет;
- основная панель при открытой задаче: `520px`;
- панель задачи: `464px`, видима только при выбранной задаче;
- переход `Сегодня -> Заметки -> Задачи` возвращает в `Сегодня`;
- подзадача создаётся, завершается и отображает прогресс `1/1`;
- таймер Focus запускается, ставится на паузу и сохраняется;
- категория звука и громкость сохраняются после перезагрузки;
- привычки показывают 28 точек истории для четырёх стартовых привычек и поддерживают смену группы.

Mobile, `390 x 844`:

- горизонтальное переполнение: нет;
- основная панель занимает `390px`;
- параметры задачи открываются отдельным полноэкранным слоем шириной `390px`;
- после закрытия параметров список задач снова занимает экран;
- нижняя модульная навигация и верхняя навигация задач остаются доступны.

## 6. Оставшиеся риски

- Встроенный браузер Codex не предоставляет `AudioContext`, поэтому фактический звук нельзя подтвердить в нём. Логика выбора, громкости и сохранения проверена; аудио нужно один раз проверить в обычном Chrome/Safari пользовательским кликом.
- Текущий storage остаётся единым JSON state. Для роста объёма заметок и совместного редактирования потребуется миграция на нормализованные таблицы, описанные в документации проекта.
- Reliability check сейчас проверяет статические контракты. Следующий инженерный шаг — добавить Playwright smoke test для создания, изменения и восстановления объектов после reload.
