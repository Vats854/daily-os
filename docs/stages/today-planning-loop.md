# Stage: Today planning loop

## Outcome

`Сегодня` становится надежным рабочим списком: показывает всё просроченное и назначенное на сегодня в понятном порядке, а выбранную задачу можно одним действием поставить в календарь. Созданный блок сохраняется сразу.

## Work layer

Implementation and verification.

## Source of truth

- `PRODUCT.md`
- `DESIGN.md`
- `docs/interface-architecture-reset.md`
- `docs/ui-calibration-audit.md`

## Scope

- Собрать Today из открытых задач с `planBucket=today`, датой сегодня или просроченной датой.
- Разделить список на `Просрочено`, `По времени`, `Остальное на сегодня`.
- Сортировать детерминированно: закрепление, время, приоритет, дата обновления.
- Показывать срок и время прямо в строке задачи.
- Добавить действие `Поставить в календарь` / `Открыть в календаре` в меню задачи.
- Сохранять связанный календарный блок немедленно через существующий `saveState()`.
- Покрыть правила Today и создание связанного блока unit-тестами.

## Non-goals

- Новый dashboard или второй timeline на экране Today.
- Recurrence, Apple Calendar и внешняя двусторонняя синхронизация.
- Переделка заметок, проектов, авторизации или визуальной системы приложения.

## Acceptance criteria

1. Просроченная открытая задача видна в Today даже без `planBucket=today`.
2. Задача с сегодняшней датой и временем находится в секции `По времени`.
3. Обычные задачи Today находятся в секции `Остальное на сегодня`.
4. Задача не дублируется между секциями.
5. Действие из карточки создаёт один связанный блок, открывает календарь и сохраняет состояние.
6. Повторное действие для уже связанной задачи открывает существующий блок вместо создания дубля.
7. Все проверки проекта и browser smoke проходят без горизонтального overflow.

## Risks

- Старые задачи могут не иметь нормализованных дат; helper обязан считать их обычными задачами Today.
- Автоматически выбранное время может пересечься с другим блоком; MVP не решает конфликты автоматически, но открывает созданный блок для правки.

## Stopping condition

Этап закончен после unit/check/browser verification и отчёта в `docs/`.
