# Daily OS / Second Brain — Product Context

## Register

Product UI / operational productivity tool. Not a landing page, not a marketing site, not a generic dashboard.

## Product promise

The user opens the app and understands:

1. what matters now;
2. what is already committed;
3. what needs review or confirmation;
4. what the AI operator changed or challenged;
5. how today connects to week and long-running projects.

## Audience

Single expert user building a personal Life OS around work search, projects, training/recovery, learning, and daily execution. The interface should feel calm, dense, and operational — closer to Linear / Weeek / Notion / Readwise / Cron than to a decorative AI dashboard.

## Primary objects

- Inbox item: raw thought or incoming input that needs interpretation.
- Task: executable action with independent planning horizon (`inbox`, `backlog`, `this_week`, `today`), workflow stage (`todo`, `in_progress`, `done`), and optional due date, time, and reminder.
- Daily plan: accepted blocks of the day and linked tasks.
- Weekly focus: strategic stake for the week, not a checkbox task.
- Project: long-running life/work focus with stage, quest, blockers, and transition criteria.
- AI action: auditable change, suggestion, challenge, or classification.
- Calendar block: an accepted time commitment with optional recurrence and system reminder.

## Navigation model

- Inbox: capture and process raw inputs.
- Today: accepted day plan and current/next action.
- Week: weekly stakes, executable tasks, backlog candidates.
- Projects: long-running projects and Hero Journey stages.
- Board: workflow pipeline over the same tasks, not a day/week duplicate or a second task store.
- Log: audit trail of AI actions.

## AI operator role

The assistant is an operator of state, not a chat decoration. It should classify, propose, challenge, require confirmation for risky changes, and leave an audit trail.

## Success criteria

- Every visible block has a job: action, decision, comparison, or editable object.
- No fake analytics, fake percentages, or decorative widgets.
- No duplicated representations of the same primary object.
- Available desktop width is used intentionally; content should not be cramped while empty columns exist.
- Inspector appears only when there is a meaningful selected object or decision to inspect.
